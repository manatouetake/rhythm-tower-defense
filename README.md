# Tower Defense (C++ / SDL2)

C++17 と SDL2 だけで実装した2Dタワーディフェンスゲームです。
ゲームエンジンを使わず、ゲームループ・当たり判定・経路移動・描画をすべて自前で実装しています。

## ゲーム内容

- 道を進む敵をタワーを設置して迎撃する
- タワー4種: Gunner(連射) / Cannon(範囲爆発) / Ice(減速) / Sniper(高威力・装甲貫通)
- 敵4種: 通常 / 高速 / タンク / **装甲兵(Sniperの貫通弾でしか倒せない)**
- 撃破報酬とウェーブクリアボーナスで資金を稼ぎ、増設・強化(最大Lv.5)・売却(60%返金)
- ウェーブごとに敵HPが指数関数的に増加(×1.17/wave)、編成も徐々に凶悪化
- 2倍速・ウェーブ自動開始に対応

## 操作

| 操作 | 内容 |
|---|---|
| ショップのカードをクリック → 空きマスをクリック | タワー設置(道の上は不可) |
| 設置済みタワーをクリック | 選択(射程表示・強化/売却ボタンが出る) |
| 右クリック / Esc | 選択解除 |
| Start Wave | 次のウェーブ開始 |

## ビルド方法

### Linux (Ubuntu/Debian)
```bash
sudo apt install g++ cmake libsdl2-dev libsdl2-ttf-dev
cmake -B build && cmake --build build
./build/towerdefense
```
CMakeを使わない場合の1行ビルド:
```bash
g++ -std=c++17 src/*.cpp $(sdl2-config --cflags --libs) -lSDL2_ttf -o towerdefense
```

### macOS
```bash
brew install sdl2 sdl2_ttf cmake
cmake -B build && cmake --build build
./build/towerdefense
```

### Windows (Visual Studio + vcpkg)
```bash
vcpkg install sdl2 sdl2-ttf
cmake -B build -DCMAKE_TOOLCHAIN_FILE=<vcpkgのパス>/scripts/buildsystems/vcpkg.cmake
cmake --build build --config Release
```

### フォントについて
システムフォント(DejaVu / Arial)を自動検出します。見つからない環境では
`assets/font.ttf` に任意のTTFフォントを置いてください。
※ UI文言を英語にしているのは、追加フォント同梱なしでどの環境でも動かすためです。

## 設計のポイント(ファイル構成)

```
src/
├── main.cpp      エントリーポイント(Gameクラスへ委譲するだけの薄い入口)
├── Config.h/.cpp ゲーム定数とタワー/敵の定義データ(データ駆動設計)
├── Entities.h    Enemy/Tower/Projectile/Particle のデータ構造(ロジックを持たないPOD)
└── Game.h/.cpp   ゲームループ・状態管理・更新・描画
```

- **データ駆動設計**: タワーや敵のステータスは `Config.cpp` の配列に集約。
  新タワーの追加は定義を1行足すだけでショップUIにも自動反映される
- **可変フレームレート対応**: `SDL_GetPerformanceCounter` で経過時間(dt)を計測し、
  全ての移動・タイマーをdt基準で計算。性能の異なるPCでも同じゲーム速度になり、
  2倍速機能もdtに係数を掛けるだけで実現
- **ID参照による安全なターゲット追跡**: 弾は敵への生ポインタではなく一意IDを保持。
  敵がvectorから削除されてもダングリングポインタが発生しない
- **erase-removeイディオム**: 死亡した敵・命中した弾の削除を `std::remove_if` で
  O(n)一括処理
- **ターゲット選択AI**: 「ウェイポイント到達数 × 大きな重み − 次の角までの残距離」を
  進行度として、射程内で最もゴールに近い敵を優先攻撃。倒せない装甲兵は狙わない

## v2: リズム連動アップデート

- **画面が拍で脈動**: 草地・道の明滅、拍頭のスクリーンシェイク、HUDの拍インジケーター(4点)とコード名表示
- **キャラの拍同期アニメ**: タワーは拍でスクワッシュ&ストレッチ、敵は交互にぴょこぴょこ跳ねる
- **Beamタワー(新)**: 8拍ごとに全基が「マウスカーソル方向」へ一斉に貫通ビーム。チャージバーが拍で満ちる
- **Gold Mine(新)**: 拍ジャストでクリックすると「PERFECT」でゴールド大量+キラキラのコイン音(BGMのコードに同期)。1小節ごとの自動収入も
- **コンボシステム**: 連続撃破でゴールドボーナス。拍ジャストの撃破は報酬1.5倍("ON BEAT!")
- **ボス**: 5の倍数ウェーブの最後に登場。生存中は地響き(拍ごとのシェイク)
- **リッチなエフェクト**: 火花・コインの放物線・浮き上がるテキスト・弾のトレイル・グラデーションビーム

実装の核は `AudioEngine` がオーディオスレッドの再生サンプル数を `std::atomic` で公開し、
ゲームスレッドがそれを「唯一の時間基準」として全演出を駆動している点(`Game::beatPhaseNow`)。
映像と音のズレが原理的に発生しない設計になっている。
